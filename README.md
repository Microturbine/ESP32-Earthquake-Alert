# ESP32 Emergency Warning System (EWS) Decoder

ESP32とFMラジオモジュール（RDA5807M）を使用して、日本国内のFM放送で運用されている**緊急警報放送（EWS）**および**緊急地震速報（EEW）**をデコード・監視するプロジェクトです。

## 概要

このシステムは、FM放送の音声信号（モノラル/ステレオ）をESP32のアナログ入力（ADC）から取り込み、信号処理によってデジタルデータを復元します。特に、自動起動信号として用いられる**64bps FSK信号**のデコードに特化しています。

## 主な機能

- **FSK復調 (Frequency Shift Keying)**: 
    - 64bpsのデータ伝送をリアルタイムにデコード。
    - **Logic 1**: 1024Hz / **Logic 0**: 640Hz を監視。
- **Goertzelアルゴリズム**: 高速フーリエ変換(FFT)より低負荷なアルゴリズムで特定周波数の強度を抽出。
- **EWSプロトコル解析**: 
    - 同期ワード（`0xB135`）を検出し、メッセージを同期。
    - 分類コード（開始信号、テスト信号、終了信号）を判別。
- **FMラジオ制御**: I2C経由でRDA5807Mの周波数や音量を動的に設定可能。

## ハードウェア構成

### 1. 構成部品
- **ESP32 開発ボード** (例: ESP32-DevKitC)
- **RDA5807M** FMラジオモジュール

### 2. ピン接続

| ESP32 ピン | RDA5807M ピン | 機能 |
| :--- | :--- | :--- |
| 3.3V | VCC | 電源 (3.3V) |
| GND | GND | グランド |
| GPIO 21 | SDA | I2C データ |
| GPIO 22 | SCL | I2C クロック |
| GPIO 34 | L-OUT (経由) | アナログ音声入力 |

## ソフトウェア解説

### デコード・ステートマシン
コード内では以下の3つの状態を遷移しながらデコードを行います：
1. **SEARCH_PREAMBLE**: 信号の存在（1/0の連続）を監視。
2. **SEARCH_SYNC**: 同期フラグ `0xB135` (10110001 00110101) を検索。
3. **DECODE_DATA**: 同期後の分類コード（4bit）などを取得・解析。

### サンプリング仕様
- サンプリング周波数: **8000Hz**
- 1ビットあたりのサンプル数: **125点** (8000Hz / 64bps)
- この窓関数に対してGoertzelフィルタを適用し、1024Hzと640Hzの優位性を比較してビット（1/0）を判定します。

## 使い方

1. お住まいの地域のNHK-FMなどの周波数に合わせて `FM_STATION` を書き換えます。
2. ライブラリマネージャーから `RDA5807` ライブラリをインストールします。
3. コードをESP32に書き込み、シリアルモニタ（115200bps）を起動します。
4. 放送内に信号が含まれる場合（試験放送時など）にデコード結果が表示されます。

## 注意事項

- 本装置は実験・学習用であり、実際の防災用としては市販の型式検定合格品を使用してください。
- 受信環境（ノイズや電波強度）によってデコード精度は大きく左右されます。
